A **Discussion message** is a note or comment added to a ticket where users can communicate, share updates, ask questions, or provide feedback related to that ticket.

A **System message** is an automatic notification generated used to inform users about changes or updates to a ticket. If any changes are made to a ticket, the system displays a system message indicating which user made the change, when was updated, and what was updated. This system message supports changes to the title, assignee, milestone, version, feature, status, priority, start and end dates, rejected status, testing, and labels. However, system messages are not generated for changes to followers, links, descriptions, or attachments.

### Why it matters / Use Cases:
- **Clear Context and Communication**: Provides clear context and communication by capturing user discussions within the ticket.
- **Change Traceability**: Ensures visibility and transparency by showing what was changed, when, and by whom.
- **Comprehensive Audit Trail**: Maintains a reliable audit trail, covering both human decisions and system-tracked updates.
- **Centralized Communication and Accountability**: Improves collaboration and accountability by centralizing all conversations and changes.

## Features
### Discussion Message
- A discussion message is displayed at the bottom of the ticket edit page.
- The sent message can be edited, replied to, shared, or deleted.

![](/img/discussion/discussion-message.png)

### System Message
- Automatically generated notifications that inform users about changes or updates to a ticket.
- Clearly indicate **who made the change**, **what was changed**, and **when the update occurred**.
- Support tracking changes to the `title`, `assignee`, `milestone`, `version`, `feature`, `status`, `priority`, `start and end date`, `rejected status` `testing`, and `labels`.
- Do not generate messages for changes related to `followers`, `linked items`, `descriptions`, or `attachments`.

## FAQs / Troubleshooting

<details>
<summary>Can a discussion message be modified?</summary>

Yes, it can be **edited**, **replied to**, **shared**, or **deleted**.

</details>

<details>
<summary>What triggers a system message?</summary>

If any changes are made to a ticket, such as updates to title, assignee, milestone, version, feature, status, priority, start and end dates, rejected status, testing, or labels then it triggers a system message.

</details>

<details>
<summary>Can system messages be manually created?</summary>

No, they are automatically generated by the system.

</details>

<details>
<summary>For which items are system messages not generated?</summary>

System messages are not generated for changes to **followers**, **linked items**, **descriptions**, or **attachments**.

</details>

<details>
<summary>Can system messages be deleted?</summary>

No, system messages are permanent to ensure traceability and auditability.

</details>

<details>
<summary>Who can post a discussion message?</summary>

Only users with access to the ticket (assignees, followers, or authorized users) can post messages.

</details>
